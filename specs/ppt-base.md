# VATOS PPT Design Guide

## Purpose

이 문서는 VATOS `DESIGN.md`의 공통 디자인 기준을 기반으로, PPT 산출물에서 자주 발생하는 레이아웃 치우침, 텍스트 넘침, 표/카드 깨짐, 슬라이드 밀도 문제를 방지하기 위한 문서 타입별 세부 규격이다.

공통 브랜드 컬러, 로고, 타이포그래피, 디자인 톤, Do/Don't, 수정 가능성 원칙은 `DESIGN.md`를 따른다.

본 문서는 공통 디자인 원칙을 반복하기보다, PPT 제작 과정에서 발생하기 쉬운 화면 구성 문제를 예방하는 데 목적이 있다.

---

## 1. Slide Layout Stability Rules

PPT 슬라이드는 기본적으로 화면 중앙 정렬을 기준으로 구성한다.  
모든 주요 콘텐츠는 슬라이드 중앙의 안전 영역 안에 배치해야 하며, 의도 없는 좌측 또는 우측 치우침을 만들지 않는다.

### 1.1 Slide Canvas Rule

- 기본 슬라이드 비율은 16:9를 기준으로 한다.
- 주요 콘텐츠는 슬라이드 중앙축을 기준으로 정렬한다.
- 표지, 섹션 구분, Closing을 제외한 일반 본문 슬라이드는 중앙 안전 영역을 사용한다.
- 전체 콘텐츠 블록이 화면 한쪽으로 치우쳐 보이지 않도록 한다.
- 좌측 정렬 레이아웃을 사용할 경우에도 콘텐츠 그룹 전체는 슬라이드 중앙 안전 영역 안에 배치한다.
- 장식 요소, 사선 레이어, 배경 패턴은 화면 바깥까지 확장할 수 있으나 본문 텍스트와 핵심 정보는 안전 영역 안에 유지한다.

### 1.2 Safe Area

- 상단, 하단, 좌우 가장자리에는 충분한 여백을 둔다.
- 핵심 텍스트, 표, 카드, 차트는 슬라이드 가장자리에 붙이지 않는다.
- 일반 본문 슬라이드의 콘텐츠 그룹은 슬라이드 너비의 약 80~86% 안에서 구성한다.
- 표와 카드 그리드는 슬라이드 중앙을 기준으로 균형 있게 배치한다.
- 표지, 섹션 구분, Closing 장표는 디자인 목적에 따라 더 넓은 배경 영역을 사용할 수 있으나 핵심 텍스트는 안전 영역 안에 둔다.

### 1.3 Alignment Rules

- 제목은 좌측 정렬할 수 있지만, 제목 블록 자체는 슬라이드 안전 영역 안에 배치한다.
- 카드, 표, 차트는 동일한 좌우 여백을 기준으로 정렬한다.
- 2열 레이아웃은 좌우 컬럼의 시각적 무게가 균형을 이루도록 구성한다.
- 3열 또는 4열 카드 레이아웃은 전체 그룹을 중앙 정렬한다.
- 표는 슬라이드 중앙에 배치하고, 좌우 여백이 비슷하게 보이도록 한다.
- 로고, 페이지 번호, 섹션 라벨은 보조 요소이며 핵심 콘텐츠 정렬 기준을 흐트러뜨리지 않는다.

### 1.4 Vertical Balance Rules

PPT 본문 슬라이드는 좌우 정렬뿐 아니라 상하 균형도 함께 고려한다.  
콘텐츠 양이 적은 슬라이드는 상단에만 몰리지 않도록, 핵심 콘텐츠 그룹을 슬라이드의 시각적 중앙 영역에 배치한다.

- 일반 본문 슬라이드에서 콘텐츠 양이 적을 경우, 카드·표·차트 그룹은 슬라이드 상단이 아니라 중앙 영역에 배치한다.
- 제목과 설명문은 상단에 배치할 수 있으나, 본문 콘텐츠 그룹은 슬라이드의 세로 중앙축을 기준으로 균형 있게 배치한다.
- 카드 3개 또는 4개만 있는 슬라이드는 카드 그룹을 화면 중앙보다 과도하게 위로 올리지 않는다.
- 하단에 주석, 단위, 페이지 번호만 있고 본문 콘텐츠가 없는 경우, 카드 그룹을 약간 아래로 내려 전체 화면의 상하 여백이 균형 있게 보이도록 한다.
- 슬라이드 하단의 빈 공간이 상단 콘텐츠 영역보다 과도하게 넓어 보이지 않도록 한다.
- 제목 영역, 본문 콘텐츠 영역, 하단 주석 영역이 각각 분리되어 보이도록 충분한 간격을 유지하되, 본문 콘텐츠가 상단에 붙어 보이면 안 된다.
- 정보량이 적은 요약 슬라이드는 카드 그룹을 세로 중앙 또는 중앙보다 약간 위에 배치한다.
- 정보량이 많은 상세 슬라이드는 상단 정렬을 사용할 수 있으나, 콘텐츠가 하단까지 자연스럽게 이어지도록 구성한다.

## 2. Content Density Rules

PPT 슬라이드는 한 장에 하나의 핵심 메시지를 전달하는 것을 기본으로 한다.  
정보를 많이 넣기보다, 제목·요약·본문의 위계를 명확히 하여 빠르게 이해되는 보고서형 구성을 우선한다.

- 일반 본문 슬라이드는 하나의 핵심 메시지를 기준으로 구성한다.
- 본문 bullet은 3~5개를 권장한다.
- 카드형 정보는 3개 또는 4개 구성을 기본으로 한다.
- 5개 이상의 항목은 표, 단계형 구조, 별도 슬라이드로 분리한다.
- 제목, 긴 본문, 표, 카드, 차트를 한 슬라이드에 모두 넣지 않는다.
- 글자 크기를 줄여서 내용을 억지로 넣지 않는다.
- 슬라이드가 복잡해질 경우 내용을 줄이는 것이 아니라, 요약 슬라이드와 상세 슬라이드로 분리한다.
- 임원 보고 또는 고객 제안용 슬라이드는 제목과 요약만 읽어도 핵심 결론이 이해되도록 구성한다.

## 3. Approved Slide Layout Types

PPT 본문 슬라이드는 아래 레이아웃 중 하나를 우선 사용한다.  
AI는 임의의 장식형 배치를 만들기보다 문서 목적에 맞는 안정적인 보고서형 레이아웃을 선택한다.

### 3.1 Title + Summary + Body

상단 제목, 핵심 요약, 본문 콘텐츠 순서로 구성하는 가장 기본적인 보고서형 레이아웃이다.

- 상단에는 슬라이드 제목을 배치한다.
- 제목 아래에는 핵심 요약문 또는 요약 카드를 배치한다.
- 본문 영역에는 bullet, 표, 카드, 차트 중 하나를 중심 콘텐츠로 배치한다.
- 보고서형 일반 본문 슬라이드에 우선 사용한다.
- 요약 영역과 본문 영역의 간격을 충분히 확보한다.

### 3.2 2 Column Layout

설명과 근거, 문제와 해결, 현황과 개선안처럼 대비되는 정보를 나란히 보여줄 때 사용한다.

- 좌측에는 배경, 설명, 문제 정의를 배치한다.
- 우측에는 수치, 결과, 표, 차트, 다이어그램을 배치한다.
- 좌우 컬럼의 시각적 무게가 균형을 이루도록 한다.
- 한쪽 컬럼에만 과도하게 많은 텍스트를 넣지 않는다.
- 좌우 컬럼 사이에는 충분한 간격을 둔다.
- 두 컬럼은 동일한 상단 기준선을 사용한다.

### 3.3 Card Grid Layout

핵심 항목, 서비스 영역, 추진 과제, 기대 효과처럼 병렬 정보를 정리할 때 사용한다.

- 3개 또는 4개 카드 구성을 기본으로 한다.
- 모든 카드는 동일한 너비와 높이를 사용한다.
- 카드 내부의 제목, 설명, 수치 위치를 통일한다.
- 카드 수가 많아지면 표 또는 별도 슬라이드로 분리한다.
- 카드마다 서로 다른 장식 요소를 넣지 않는다.
- 카드 간 간격은 균일하게 유지한다.

### 3.4 Table Focus Layout

비교, 현황, 점검 결과, 일정, 튜닝 결과처럼 표가 핵심 정보인 경우 사용한다.

- 표가 핵심인 슬라이드는 표를 중앙에 배치한다.
- 표 위에는 표를 읽는 기준이나 핵심 요약을 한 줄로 작성한다.
- 표와 함께 카드, 차트, 긴 본문을 동시에 배치하지 않는다.
- 표가 작아져 읽기 어려우면 슬라이드를 분리한다.
- 상세 표는 별도 슬라이드 또는 부록으로 분리할 수 있다.

### 3.5 Chart Focus Layout

수치 변화, 추세, 비교 결과, 구성비를 시각적으로 보여줄 때 사용한다.

- 차트가 핵심인 슬라이드는 차트를 가장 큰 정보 블록으로 배치한다.
- 차트 주변에는 핵심 인사이트 1~3개만 배치한다.
- 차트와 표를 같은 비중으로 나란히 배치하지 않는다.
- 차트 제목은 수치가 말하는 결론을 담아 작성한다.
- 차트 하단 또는 우측에는 필요한 경우 간단한 해석 문구를 배치한다.

## 4. Text Overflow & Wrapping Rules

PPT는 화면 크기가 제한되어 있으므로 한글 문장의 줄바꿈과 텍스트 넘침을 반드시 고려한다.

- 제목은 1~2줄 안에서 작성한다.
- 제목이 2줄을 초과하면 문장을 줄이거나 부제/본문으로 분리한다.
- 카드 제목은 1줄을 기본으로 하며, 최대 2줄을 넘지 않는다.
- 카드 설명은 2~3줄 이내로 작성한다.
- 본문 bullet은 한 항목당 1~2줄을 권장한다.
- 표 셀 안의 문장은 줄바꿈을 고려해 충분한 행 높이를 확보한다.
- 텍스트가 박스 밖으로 넘치지 않도록 텍스트 박스 너비를 우선 확보한다.
- 글자 크기를 과도하게 줄여 해결하지 않는다.
- 긴 문장은 핵심어와 설명을 분리하여 작성한다.
- 한글과 영문이 섞인 문장은 줄바꿈 위치가 어색하지 않도록 문장 길이를 조정한다.
- 수치, 단위, 제품명, 고객사명은 줄바꿈으로 분리되어 의미가 깨지지 않도록 한다.

## 5. PPT Table Stability Rules

표 정렬 기준은 `DESIGN.md`의 Table Alignment Rules를 따른다.  
이 섹션에서는 PPT 슬라이드 안에서 표가 깨지거나 읽기 어려워지는 문제를 방지한다.

- 표는 슬라이드 중앙 안전 영역 안에 배치한다.
- 표가 슬라이드 가장자리까지 밀리지 않도록 한다.
- 긴 설명 컬럼은 너비를 넓게 배정하고, 수치 컬럼은 좁게 배정한다.
- 표 안의 텍스트가 셀 밖으로 넘치지 않도록 행 높이를 확보한다.
- 표가 8행 또는 6열을 초과하면 요약 표와 상세 표를 분리한다.
- 표가 작아져 읽기 어려운 경우 별도 상세 슬라이드로 분리한다.
- 표 상단에는 표를 해석하는 기준이나 핵심 요약을 짧게 배치한다.
- 표 하단에는 필요한 경우 단위, 기준일, 출처, 주석을 작게 배치한다.
- 표 안에 지나치게 많은 색상 강조를 넣지 않는다.
- 강조가 필요한 행이나 셀은 1~3개 이내로 제한한다.

## 6. PPT Output Review Checklist

PPT 산출 전 아래 항목을 확인한다.

- [ ] 기본 슬라이드 비율이 16:9로 구성되었는가?
- [ ] 핵심 콘텐츠가 슬라이드 중앙 안전 영역 안에 배치되었는가?
- [ ] 본문 콘텐츠가 좌측 또는 우측으로 의도 없이 치우치지 않았는가?
- [ ] 제목, 카드, 표, 차트의 좌우 기준선이 정리되어 있는가?
- [ ] 한 슬라이드에 하나의 핵심 메시지만 담겼는가?
- [ ] 본문 bullet이 과도하게 많지 않은가?
- [ ] 카드 수가 3개 또는 4개 기준을 크게 벗어나지 않는가?
- [ ] 텍스트가 박스 밖으로 넘치지 않는가?
- [ ] 제목이 1~2줄 안에서 읽히는가?
- [ ] 카드 설명이 2~3줄 이내로 정리되었는가?
- [ ] 표가 너무 작아져 읽기 어려운 상태가 아닌가?
- [ ] 표가 슬라이드 가장자리까지 밀리지 않았는가?
- [ ] 차트, 표, 카드가 한 슬라이드에 과도하게 혼합되지 않았는가?
- [ ] 공통 브랜드 컬러, 로고, 타이포그래피, Do/Don't 기준은 `DESIGN.md`를 따랐는가?
- [ ] PPT 요소가 PowerPoint에서 수정 가능한 형태로 구성되었는가?
